    banking v0.1.0 API Reference

    Modules

    Banking

    The entry point to define macros and common behavior of the Banking application (if any).

    Banking.AccountManagement

    Provide functions to manage accounts and users

    Banking.AccountManagement.EmailVerification

    Provides functions to generate email verification code and their validation

    Banking.AccountManagement.Password

    Is responsible for hashing the original password and decoded it.

    Banking.Admin

    Provides the functions to verify admin identity.

    Banking.Bank

    Processes all the financial operations

    Banking.Bank.KPI

    Provides many different indicators about the bank operations

    Banking.Bank.KPI.AmountVolume

    Reports the total amount volume that was transferred or withdraw

    Banking.Bank.Seeds

    Provides the initial data necessary for the system to work.

    Banking.Bank.SpecialAccounts

    Keep track of the special accounts that are used as registers to share the same interface of common transactions. This solution might move to a proper table in the database if we starting to interact with different banks and sources.

    Banking.Bonus

    Handles bonus and promotions

    Banking.Repo

    The repository to interact with the PostgreSQL database used in the Banking Application.

    Banking.Session

    Handles the user session by tracking the IP and device used.

    Banking.Session.Guardian

    Defines how to serialize and deserialize sessions from JWT

    Banking.Session.Guardian.Plug
    BankingWeb

    The entry point for defining your web interface, such as controllers, views, channels and so on.

    BankingWeb.EmailVerificationController
    BankingWeb.Endpoint

    Defines Banking endpoint.

    BankingWeb.ErrorHelpers

    Conveniences for translating and building error messages.

    BankingWeb.ErrorView
    BankingWeb.Gettext

    A module providing Internationalization with a gettext-based API.

    BankingWeb.GraphQL.Plugs.Context

    Prepares the context to allow Absinthe to handle authentication and authorization

    BankingWeb.GraphQL.Plugs.Pipeline

    Plug that performs a pipeline of guardian operations to parse the token from the Authorization header and load the resource if any

    BankingWeb.GraphQL.Schema

    Defines the main GraphQL's schema used by absinthe.

    BankingWeb.LayoutView
    BankingWeb.PageController
    BankingWeb.PageView
    BankingWeb.Router
    BankingWeb.Router.Helpers

    Module with named helpers generated from BankingWeb.Router.

    BankingWeb.UserSocket
    ComboWeb.GraphQL.SessionErrorHandler

    Behaviour for creating error handlers for Guardian.Plug.Pipeline.
